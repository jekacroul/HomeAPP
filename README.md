# Техническое задание (MVP)

## Приложение: «Где работать» (для удалёнщиков)

---

## 1. Цель продукта

Создать приложение, которое помогает удалённым работникам быстро находить удобные места для работы (кафе, коворкинги и т.д.) с учетом ключевых факторов: Wi-Fi, уровень шума, наличие розеток и возможность длительного пребывания.

---

## 2. Целевая аудитория

* Удалённые сотрудники
* Фрилансеры
* Студенты
* Digital-номады

---

## 3. Основные сценарии использования

### 3.1 Быстрый поиск места

Пользователь открывает приложение и:

* Видит карту или список мест рядом
* Фильтрует по параметрам
* Выбирает место

### 3.2 Проверка перед выходом

* Смотрит актуальные отзывы
* Проверяет загруженность

### 3.3 Добавление нового места

* Пользователь добавляет новое место
* Указывает параметры

### 3.4 Оставление отзыва

* Оценивает место по критериям
* Добавляет комментарий

---

## 4. Основной функционал (MVP)

### 4.1 Авторизация

* Email / Google (минимум)

### 4.2 Карта и список мест

* Отображение мест на карте
* Список с краткой информацией

### 4.3 Карточка места

Содержит:

* Название
* Фото
* Рейтинг
* Параметры:

  * Wi-Fi (да/нет/скорость)
  * Уровень шума (тихо/средне/шумно)
  * Розетки (много/мало/нет)
  * Можно ли сидеть долго (да/нет)
* Отзывы

### 4.4 Фильтры

* Wi-Fi
* Уровень шума
* Розетки
* Подходит для работы с ноутбуком

### 4.5 Добавление места

Форма:

* Название
* Адрес
* Фото
* Параметры

### 4.6 Отзывы

* Текст
* Оценки по параметрам

---

## 5. Дополнительный функционал (после MVP)

* Проверка скорости Wi-Fi (интеграция теста)
* Загруженность в реальном времени
* Избранные места
* Рекомендации
* Геймификация (очки за отзывы)

---

## 6. Нефункциональные требования

* Быстрая загрузка (<2 сек)
* Простота интерфейса
* Работа на iOS / Android (или Web MVP)

---

## 7. Технологический стек (рекомендация)

### Frontend

* React Native / Flutter (мобилка)
  или
* React (Web MVP)

### Backend

* Node.js (NestJS / Express)

### База данных

* PostgreSQL

### Карты

* Google Maps API / Mapbox

### Хостинг

* AWS / Vercel / Firebase

---

## 8. Структура данных (упрощенно)

### Place

* id
* name
* address
* lat
* lng
* wifi
* noise_level
* sockets
* stay_duration_allowed

### Review

* id
* place_id
* user_id
* rating
* comment

### User

* id
* email

---

## 9. Метрики успеха

* Количество добавленных мест
* Количество отзывов
* Retention (возврат пользователей)

---

## 10. План разработки (MVP)

### Неделя 1

* Проектирование
* База данных
* Backend API

### Неделя 2

* UI (список + карточка)
* Добавление мест

### Неделя 3

* Отзывы
* Фильтры
* Деплой

---

## 11. Риски

* Нет контента (пустая база)
* Нет пользователей
* Слабая ценность без данных

---

## 12. Как снизить риски

* Начать с одного города
* Самому добавить 50–100 мест
* Привлечь первых пользователей вручную

---

## 13. MVP-ограничение

Важно:

* НЕ делать социальную сеть
* НЕ делать сложную аналитику
* НЕ делать всё сразу

Фокус: быстро дать ценность пользователю
